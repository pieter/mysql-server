Upgrading Berkeley DB Applications

Release 3.2: DBENV->set_mutexlocks

Previous Berkeley DB releases included the db_env_set_mutexlocks interface, intended for debugging, that allows applications to always obtain requested mutual exclusion mutexes without regard for their availability. This interface has been replaced with DBENV->set_mutexlocks, which provides the same functionality on a per-database environment basis. Applications using the old interface should be updated to use the new one.
